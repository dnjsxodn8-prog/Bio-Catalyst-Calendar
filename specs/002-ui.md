# Spec 002: UI

**상태:** Draft
**작성일:** 2026-04-27
**관련 요구사항:** 11번(페이지 4개), 12번(헤더 버튼 제거)
**참고 디자인:** 사용자가 만든 [bio-catalyst-calendar.vercel.app](https://bio-catalyst-calendar.vercel.app/) (스크린샷 4장)

## 목표

`bio-catalyst-calendar.vercel.app`의 UI를 참고하여 4개 페이지 구조의 다크모드 대시보드를 만든다. 데이터는 `src/data.generated.json`(396개 종목 + 44 카탈리스트 + 24 학회)에서 로드. UI에서 추가/삭제 기능은 모두 제거.

## 비-목표

- 다중 사용자
- 추가/편집/삭제 UI 기능 (모두 Claude Code 통해서만)
- localStorage (참고 사이트는 localStorage 기반이지만 우리는 git+md 기반)
- Live API 페이지 (HANDOFF 11번)

## 페이지 구조 (4개)

```
대시보드 / 종목 / 카탈리스트 / 학회
```

❌ Live API 탭 — 제거 (HANDOFF 11)
❌ 헤더 버튼 4개 (종목 추가, 이벤트 추가, CSV, 초기화) — 제거 (HANDOFF 12)
✅ 검색창과 필터만 각 페이지 내부에 잔존
❌ 푸터의 "localStorage 자동 저장" — 제거. 대신 "빌드 시각: YYYY-MM-DD HH:mm" 표시.

## 페이지별 사양

### 1. 대시보드

**상단 헤더:**
- 좌측: 📈🧬 **Biotech Catalyst Dashboard** (제목)
  - 부제: "100+ 종목 · 2026 PDUFA · 학회 · 임상 readout"
- 우측: (비어있음 — 4버튼 모두 제거)

**탭 네비게이션:** 대시보드 / 종목 / 카탈리스트 / 학회 (활성 탭은 그린 underline)

**4개 통계 카드** (그리드 4열, 모바일에서 2열):
- 30일 이내 임박 카탈리스트 (count)
- 90일 이내 단기 카탈리스트
- 전체 PDUFA (FDA 결정 대기)
- 임상 readout 전체 예정

**임박한 카탈리스트 (60일 이내)**
- 카드 배경 박스 안에 행 리스트
- 각 행: `[D-N 배지]` `[날짜]` `[ticker (그린)]` `[이벤트명]` ··· `[type 배지]` `[펼침 ▼]`
- 펼치면: 해당 카탈리스트의 약물·적응증·Phase + 종목 디테일 (종목 페이지와 동일)

**최근 결과 (지난 30일)**
- 임박 리스트와 동일한 행 디자인. D+N 배지(회색)로 과거 표기.

**Modality 분포 + 치료 영역 분포** (2단 그리드)
- 모집단: **60일 이내 임박 카탈리스트의 종목들** (전체 396종목 X)
- 수평 막대 차트 (직접 SVG)
- 각 항목: `라벨 ─── (count) (%)`
- Modality는 그린, 치료 영역은 보라

### 2. 종목 페이지

**상단 필터 바:**
- 🔍 검색 (ticker/회사명 부분일치)
- Modality 드롭다운 (전체 + 10개 enum)
- 영역 드롭다운 (전체 + 데이터에 등장하는 모든 area)
- 정렬 드롭다운 (시총↓/시총↑/D-day↑)

**N / N 종목** 표시 (필터 결과 / 전체)

**종목 행 리스트:** 각 행:
```
[★] [ticker (그린)] [회사명]                          [시총] [recommendation 배지] [▼]
                  [부제: areas]
                  [임박 카탈리스트 한 줄 요약]
```

**펼친 디테일** (스크린샷 2번 참고):
1. **카탈리스트 박스** (상단 강조)
   - 약물명, 적응증, Phase
2. **기업 프로필** (아이콘 + 라벨 그리드)
   - 시가총액, 최근 매출, 핵심 활용중인 기술, 주요 적응증 포트폴리오, 주요 파트너/고객, 매출 구조, 자체 판매 능력, 상업화 단계
3. **임상 정보** (있을 때만)
   - 임상 디자인, 타겟 질환, 기존 치료제 & 차별점, 최 사전 공개 임상 데이터
4. **약물 정보** (있을 때만)
   - Modality, 작용 기전 (MOA), 주요 논문/참고문헌
5. **메모**
6. **액션 버튼은 제거** (HANDOFF 12)

### 3. 카탈리스트 페이지

**상단 필터 바:**
- 🔍 검색
- type 드롭다운

**N / 44 이벤트** 표시 + "▼ 과거 보기 (N)" 토글

**행 리스트:** 시간순 (가까운 날짜 위)
```
[D-N 배지] [날짜] [ticker] [이벤트명]                            [type 배지] [▼]
```

**펼친 디테일:** 종목 페이지와 거의 동일 + 카탈리스트 메타(약물·적응증·Phase·sources)

### 4. 학회 페이지

**상단 필터 바:**
- 🔍 검색
- 영역 드롭다운

**N / 24 학회** 표시 + "▼ 과거 보기 (N)" 토글

**카드 리스트:**
```
[D-N 배지]  [학회명 (블루)]                                        [Tier N 배지]
            [날짜 범위] · [위치] · [영역]
[메모: 설명]
```

학회는 펼치는 디테일 없음 (참고 사이트와 동일 — 정보가 단순).

## 빈 깡통 종목 처리

396개 중 330개는 빈 깡통 (modality=Other, body 모두 "정보 미입력"). 종목 페이지 디폴트 표시 시 노이즈 큼.

**옵션:**
- (A) 디폴트 필터 "실데이터만" 활성 (mcap 기준 또는 modality !== 'Other' 조건)
- (B) 그대로 다 표시. 사용자가 시총 정렬로 자연 정리.
- (C) 두 탭 분리: "추적 중" / "전체"

**디폴트 추천:** (B) + 시총 내림차순 정렬. 빈 깡통도 시총 정보는 있어서 자연스럽게 하단으로. 종목 페이지 행 디자인에서 본문이 모두 "정보 미입력"이면 펼침 화살표 흐리게.

## 기술 스택

- React 19 (Vite 기본)
- 상태관리: useState만. Redux/Zustand 불필요 (1인용).
- 라우팅: 단순 useState 탭 전환 (4개 페이지뿐이라 react-router 오버킬)
- 차트: Recharts 추천 (또는 직접 SVG 막대바)
- CSS: Tailwind v3 추천. v4도 가능하나 v3가 안정.
- 아이콘: lucide-react
- 날짜 계산: 자체 D-day 헬퍼 (라이브러리 불필요)

설치:
```bash
npm install -D tailwindcss@^3 postcss autoprefixer
npm install recharts lucide-react
```
